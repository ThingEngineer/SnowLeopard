# SnowLeopard 0.1.0-dev

## Highlights

- Establishes the first full SnowLeopard firmware baseline on ESP32-C3.
- Delivers local-first refrigeration control with browser UI and OLED runtime status.
- Includes history tracking, optional settings authentication, and password-reset gesture support.
- Introduces GitHub-hosted OTA metadata checks and release portal foundations.

## Release notes

This development release captures the initial end-to-end SnowLeopard feature set, not just OTA plumbing. It is the first version that brings up provisioning, control, monitoring, settings, and history together with the first GitHub-based firmware update path.

### Included in this baseline

- Provisioning AP mode with STA fallback and `snowleopard.local` access in STA mode.
- Relay control with AUTO, MANUAL_ON, and MANUAL_OFF modes.
- Setpoint, on/off delta, and compressor lockout controls with safety clamping.
- Internal/external sensor reads plus per-sensor calibration offsets.
- Temperature alarm thresholds with audible buzzer output and alarm test action.
- OLED runtime display with selectable layouts and high-visibility alarm screen.
- Web surfaces for status (`/`), settings (`/settings`), and history (`/history`).
- Optional settings-password protection with physical password-reset gesture.
- Reboot-survivable blended temperature history and relay-overlay charting.
- OTA manifest check and queued update flow with GitHub Pages-hosted release metadata.

## Operator notes

- Treat this build as the baseline reference for initial field validation.
- For production OTA releases, update `include/firmware_version.h` and `release-data/current.json` together.
- Always publish release notes and firmware assets for the same version tag.
