# SnowLeopard 0.1.1

## Highlights

- Promotes the 0.1.0-dev baseline into the first production-intended OTA release.
- Replaces development placeholders with a real tagged firmware asset and release metadata.
- Hardens OTA security with pinned GitHub trust roots and SHA-256 verification.
- Improves on-device update feedback with clearer progress and verification states.

## Release notes

`0.1.1` is a delta release on top of the `0.1.0-dev` baseline. It keeps the same core control, settings, history, and UI capabilities while making the OTA path production-ready.

### What changed from 0.1.0-dev

- Published a real GitHub Release firmware artifact for direct OTA download.
- Strengthened transport trust and payload validation (pinned trust roots + SHA-256 checks).
- Improved Firmware Update UX in `/settings` with better progress and validation messaging.
- Finalized the public GitHub Pages release surface for release notes and manifest consumption.

## Operator notes

- After the tagged release is built, replace the placeholder `sha256` and `size` values in `release-data/current.json` with the real artifact metadata.
- Confirm GitHub Pages is enabled in repository settings with source set to GitHub Actions.
- Test `/settings` on a device running `0.1.0-dev` and confirm that `0.1.1` appears as the latest version.
